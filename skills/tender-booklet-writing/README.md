### 1. 准备输入材料

| 材料类型 | 必需性 | 说明 |
|----------|--------|------|
| 投标方案册子PDF/设计稿 | ✅ 必需 | 需要撰写说明的目标画册 |
| 招标文件（技术评分表） | ✅ 必需 | 用于建立「评分项-章节」对应关系 |
| 策划方案文档 | ✅ 必需 | 已完成的策划篇、空间规划等核心方案 |
| 甲方设计任务书 | ⭕ 可选 | 补充需求理解 |

### 2. 阅读技能指南

详细阅读 `SKILL.md`，了解：
- 文档结构规范
- 页面说明撰写原则（价值导向、承上启下、专业表达等）
- 语言风格规范与禁用表达
- 评分项响应映射方法

### 3. 按流程撰写

```
阶段1：材料梳理 → 建立页面清单、评分项映射
    ↓
阶段2：结构设计 → 划分章节、编写导引页
    ↓
阶段3：逐页撰写 → 按规范撰写每页说明
    ↓
阶段4：汇报优化 → 编制速查表、标注时长
    ↓
阶段5：一致性检查 → 核心概念、评分覆盖、表达规范
```

## 撰写核心原则

### ✅ 正确示范

```markdown
### P23-24｜第一幕「迎」效果图与展项分析
光之序章空间以时光长卷（滑轨屏）与企业故事墙（LED）双线并置，
通过光廊形态引导访客进入，材质选用雾面金属与透光材料呼应棱镜主题。
该区域承担在最短时间内建立品牌信任的核心任务，为后续深度探索奠定情感基础。
```

### ❌ 错误示范

```markdown
### P23-24｜效果图
这一页展示了我们设计的第一个空间，非常好看，
用了很多高科技的材料。这个空间很重要。
```

### 核心差异

| 维度 | ❌ 错误 | ✅ 正确 |
|------|--------|--------|
| 内容 | 描述画面本身 | 解读设计意图与价值 |
| 语气 | 口语化、主观评价 | 正式书面语、客观表述 |
| 逻辑 | 孤立描述 | 承上启下、逻辑串联 |
| 概念 | 无概念呼应 | 核心概念贯穿全文 |

## 常见问题

### Q: 页面说明和设计稿上的文字有什么区别？

A: 设计稿上的文字是给观众看的展示内容，页面说明是给讲解人员和甲方看的解读文档。页面说明需要解释「为什么这样设计」、「这页在整体逻辑中的位置」，而不是复述画面内容。

### Q: 每页说明应该写多长？

A: 建议1-3句话，100-200字。重点页面可适当加长，但避免冗长。

### Q: 如何确保不遗漏评分项？

A: 撰写前先建立「评分项→章节→页码」映射表，撰写后检查每个评分项是否都有对应章节覆盖。

### Q: 汇报时长如何分配？

A: 建议分配比例：
- 项目理解与定位：15-20%
- 核心理念与叙事：20-25%
- 空间效果与展项：35-40%
- 技术方案：10-15%
- 团队业绩与服务：5-10%
