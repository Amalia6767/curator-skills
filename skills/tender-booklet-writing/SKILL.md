# 投标方案册子页面说明撰写

## 技能概述

本技能用于为**投标方案册子（画册）**撰写专业的页面说明文档。该文档不是册子本身的设计内容，而是对每一页内容的**文字解读与撰写要点说明**，主要用于：

1. **汇报讲解**：为讲解人员提供每页的核心话术与重点
2. **甲方阅读**：帮助甲方快速理解方案逻辑与价值主张
3. **团队沟通**：为设计团队提供明确的内容方向指引

---

## 一、输入材料要求

### 1.1 必需输入

| 材料类型 | 说明 |
|----------|------|
| **投标方案册子PDF/设计稿** | 需要撰写说明的目标画册，包含所有页面 |
| **招标文件** | 尤其是技术评分表部分，用于建立"评分项-章节"对应关系 |
| **策划方案文档** | 已完成的策划篇、空间规划等核心方案文档 |

### 1.2 可选输入

- 甲方设计任务书/需求简报
- 竞品分析报告
- 汇报时间限制要求

---

## 二、文档结构规范

### 2.1 整体框架

```markdown
# [项目名称]投标画册｜页面说明文档

> **用途**：本文档为投标画册各页撰写说明文字，可用于汇报讲解、
> 甲方阅读理解及后期方案沟通。

---

## 第一章｜[章节名称]

### P[起]-[止]｜[页面标题]
[说明正文]

---

## 第二章｜[章节名称]
...

---

## 附录｜汇报要点速查

| 页码 | 核心要点 | 建议时长 |
|-----|---------|---------|
| ... | ...     | ...     |

**总汇报时长建议**：完整版XX分钟｜精简版XX分钟
```

### 2.2 章节划分原则

1. **与评分项对应**：每章应明确响应招标评分表的具体分项
2. **逻辑递进**：章节顺序应符合阅读/汇报的认知递进逻辑
3. **标注分值**：章节标题后注明对应评分项编号与分值

**示例**：
```markdown
## 第二章｜策展与规划（响应评分项1.1-1.2，共9分）
```

## 三、页面说明撰写规范

### 3.1 基本格式

```markdown
### P[页码]｜[页面标题]
[说明正文，1-3句话，100-200字]
```

### 3.2 撰写原则

| 原则 | 说明 | 示例 |
|------|------|------|
| **价值导向** | 突出"为什么做"而非"做了什么" | ✗ "本页展示了企业发展历程"<br>✓ "时光长卷以叙事化方式呈现企业价值积淀，在最短时间内建立品牌信任" |
| **承上启下** | 说明该页在整体逻辑中的位置 | "该区域承担在最短时间内建立品牌信任的核心任务，为后续深度探索奠定情感基础" |
| **专业表达** | 使用正式、非口语化的书面语言 | ✗ "这一页特别重要"<br>✓ "本页明确各章节内容与招标评分项的对应关系" |
| **概念呼应** | 将核心概念贯穿于说明文字 | 如"棱镜式接待"、"千人千面"、"四幕剧场"等 |
| **可验证性** | 涉及可量化指标时给出具体数据 | "400㎡空间按四幕剧场结构进行功能分区：第一幕约100㎡→第二幕约120㎡..." |

### 3.3 不同类型页面的撰写重点

| 页面类型 | 撰写重点 |
|----------|----------|
| **封面/标题页** | 项目定位、核心概念金句 |
| **目录/导引页** | 响应评分项对照、结构索引价值 |
| **分析理解页** | 洞察深度、独特视角、差异化认知 |
| **方案概念页** | 核心理念阐释、概念的独创性与适配性 |
| **效果图页** | 空间设计语言、材质工艺、与概念的呼应 |
| **技术方案页** | 技术创新点、可落地性、评分响应 |
| **团队/业绩页** | 资质亮点、项目匹配度 |
| **封底页** | 合作愿景、情感共鸣收尾 |

## 四、评分项响应映射

### 4.1 建立映射表

在撰写前，先建立「评分项→章节→页码」的完整映射：

| 评分项 | 分值 | 考察内容 | 对应章节 | 页码范围 |
|--------|------|----------|----------|----------|
| 1.1 需求理解 | X分 | ... | 第二章 | P5-P14 |
| ... | ... | ... | ... | ... |

### 4.2 章节导引页要求

每章开始应有**导引页说明**，格式如下：

```markdown
### P[页码]｜章节导引页
[章节名]篇聚焦[评分考察点1]（X分）、[评分考察点2]（X分）两大核心议题，
[本章价值定位说明]。
```

## 五、语言风格规范

### 5.1 风格定位：专业、凝练、有策略视角

| 维度 | 要求 |
|------|------|
| **语气** | 正式书面语，避免口语化表达 |
| **句式** | 主动语态为主，句式简洁有力 |
| **用词** | 专业术语准确，核心概念统一 |
| **长度** | 每页说明控制在1-3句话，100-200字 |

### 5.2 禁用表达清单

| 禁用表达 | 替代表达 |
|----------|----------|
| "这一页展示了..." | "本页呈现/阐述/明确..." |
| "非常重要" | 直接说明重要性的具体体现 |
| "我们的方案" | "本方案" / 具体的方案名称 |
| "做了XX" | "实现/构建/形成XX" |
| "特别好看" | 描述具体的设计语言与效果 |

### 5.3 核心概念列表维护

撰写前梳理项目的核心概念，确保全文一致使用：

```markdown
## 本项目核心概念速查
- **核心理念**：[如：棱镜式接待]
- **叙事结构**：[如：四幕剧场 - 迎·绘·聚·续]
- **设计语言**：[如：液态叙事]
- **技术亮点**：[如：企业自研大模型]
- **价值主张**：[如：千人千面、可自主迭代的3.0数智展厅]
```

## 六、汇报速查表编制

### 6.1 速查表格式

文档末尾应附「汇报要点速查」表格：

```markdown
## 附录｜汇报要点速查

| 页码 | 核心要点 | 建议时长 |
|-----|---------|---------|
| P6-14 | 企业理解与需求定位 | 4分钟 |
| P15-19 | 核心理念与叙事逻辑 | 5分钟 |
| ... | ... | ... |

**总汇报时长建议**：完整版XX分钟｜精简版XX分钟
```

### 6.2 时长分配原则

| 内容类型 | 占比建议 |
|----------|----------|
| 项目理解与定位 | 15-20% |
| 核心理念与叙事 | 20-25% |
| 空间效果与展项 | 35-40% |
| 技术方案 | 10-15% |
| 团队业绩与服务 | 5-10% |

## 七、工作流程

### 阶段1：材料梳理

1. 阅读投标册子PDF/设计稿，建立页面清单
2. 对照招标评分表，建立「评分项→页码」映射
3. 回顾策划方案核心文档，提取核心概念

### 阶段2：结构设计

1. 划分章节，与评分大项对应
2. 确定每章包含的页码范围
3. 编写章节导引页说明

### 阶段3：逐页撰写

1. 按章节顺序逐页撰写说明
2. 确保价值导向、承上启下、专业表达
3. 核心概念复用检查

### 阶段4：汇报优化

1. 编制汇报速查表
2. 标注建议时长
3. 提炼每部分核心要点

### 阶段5：一致性检查

- [ ] 所有评分项均有对应章节覆盖
- [ ] 核心概念全文统一使用
- [ ] 无口语化/不规范表达
- [ ] 承上启下逻辑通顺
- [ ] 汇报时长合理分配
