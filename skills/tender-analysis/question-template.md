# 投标文件疑问梳理模板

在答疑会议前，使用此模板系统梳理所有疑问细项。

## 疑问梳理原则

1. **全面性**：不遗漏任何模糊、不清晰或有歧义的条款
2. **针对性**：每个疑问都要有明确的文件引用
3. **重要性分级**：区分关键疑问和一般疑问
4. **保护权益**：重点关注可能不利于乙方的条款
5. **可操作性**：疑问表述要清晰，便于甲方理解和回答

## 疑问分类标准

- **🔴 关键疑问**：影响投标决策、方案设计或报价，必须澄清
- **🟡 重要疑问**：影响实施计划、技术选型或商务条款，建议澄清
- **🟢 一般疑问**：有助于优化方案或避免误解，可选澄清

## 疑问表述规范

### 标准格式

```
【疑问编号】Q[X]
【疑问类别】🔴关键疑问 / 🟡重要疑问 / 🟢一般疑问
【文件位置】第X章第X条 / 第X页第X段
【原文引用】"[招标文件原文]"
【疑问内容】[具体说明哪里不清晰、有歧义或不利于乙方]
【可能影响】[说明如果不澄清可能造成的影响]
【建议方案】[如有，提出建议的解决方案或澄清方向]
```

### 表述要求

1. **明确引用**：必须指出具体的文件位置和原文
2. **具体说明**：说明疑问的具体内容，不要泛泛而谈
3. **影响分析**：说明疑问可能造成的影响，体现重要性
4. **建设性**：如有可能，提出建议的解决方案

### 好的疑问示例

✅ **好的疑问：**
```
【疑问编号】Q1
【疑问类别】🔴关键疑问
【文件位置】第三章第5条
【原文引用】"系统需支持10000+并发用户，响应时间<2秒"
【疑问内容】响应时间<2秒的具体定义不明确：
1. 是在什么负载条件下测试？（10000并发还是其他？）
2. 是平均响应时间还是最大响应时间？
3. 测试环境是什么？（硬件配置、网络环境等）
【可能影响】影响技术方案设计、性能测试标准和报价
【建议方案】建议明确测试环境、测试方法和指标定义
```

❌ **不好的疑问：**
```
【疑问编号】Q1
【疑问内容】技术要求不清楚
【可能影响】影响方案设计
```

## 答疑会议准备

### 会议前准备

- [ ] 疑问清单已按重要性排序
- [ ] 关键疑问已标注，优先提问
- [ ] 每个疑问都已明确文件引用
- [ ] 已准备建议方案（如适用）
- [ ] 已准备记录工具，记录甲方回复

### 会议中注意事项

1. **按重要性提问**：先问关键疑问，再问重要疑问
2. **明确记录**：详细记录甲方的回复，特别是关键问题的回复
3. **追问确认**：如回复仍不清晰，及时追问
4. **书面确认**：重要回复建议要求甲方书面确认

### 会议后处理

1. **整理回复**：将甲方回复整理到疑问清单中
2. **更新分析**：根据回复更新投标文件分析文档
3. **调整方案**：根据回复调整方案设计方向
4. **识别新信息**：识别回复中的新要求或新信息
