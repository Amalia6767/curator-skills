# 展陈大纲生成 Skill

本 skill 用于根据策划方案生成展陈大纲，以3页PPT格式呈现，便于直接用于方案汇报或PPT制作。

## 文件结构

```
exhibition-outline/
├── SKILL.md          # 核心技能说明与工作流程
└── README.md         # 本文件
```

## 输入

### 必需输入
- **策划方案**（Markdown格式）：通常来自 `proposal-design` skill 的输出
  - 例如：`投标方案_策划篇.md`

### 可选输入
- **展陈主题推导报告**：用于理解主题的推导逻辑
- **展陈大纲样本**：用于学习优秀展陈大纲的结构和表达方式

## 输出

生成 `展陈大纲_PPT版.md` 文件，包含3页PPT内容：

1. **第一页：展陈主题**
   - 主标题、副标题
   - 主题核心解读（一句话概括、空间维度、时间维度、统一理念）
   - 核心理念
   - 视觉元素建议

2. **第二页：诠释与过渡**
   - 主题诠释（核心问题、核心答案）
   - 叙事逻辑（叙事结构、情感曲线）
   - 空间规划策略（空间布局、动线设计）
   - 视觉元素建议

3. **第三页：详细的展陈大纲（要点版）**
   - 按区域组织（入口区、过渡区、业务区、科技区、体验区、出口区等）
   - 每个区域包含：情感目标、核心内容模块
   - 每个模块包含：展示形式、核心信息、叙事功能
   - 展陈大纲总结（总览、叙事主线）

## 使用场景

- **方案汇报**：向甲方或内部团队汇报展陈大纲
- **PPT制作**：直接用于制作展陈大纲PPT
- **方案深化**：作为展陈设计的起点，后续可进一步细化
- **快速沟通**：快速传达展陈的核心主题、叙事逻辑和内容结构

## 工作流程

1. **提取展陈主题**：从策划方案中提取主题、副标题、核心理念
2. **分析叙事逻辑**：识别叙事结构、情感曲线、核心问题和答案
3. **规划空间布局**：提取空间规划信息，理解各区域功能定位
4. **组织展陈大纲**：按区域组织内容，每个模块说明展示形式、核心信息、叙事功能
5. **优化与检查**：确保结构清晰、要点突出、信息完整

## 生成原则

- **结构清晰**：三页内容层次分明，逻辑递进（主题→诠释→大纲）
- **要点突出**：每页核心要点不超过5个，避免信息过载
- **便于执行**：展示形式明确，空间信息完整，叙事功能清晰
- **参考样本**：学习优秀展陈大纲的叙事驱动、情感曲线、模块化设计等特点

## 与其他 skill 的关系

- `proposal-design`：生成策划方案（输入来源）
- `exhibition-outline`：根据策划方案生成展陈大纲（本skill）
- `proposal-interpretation`：将策划方案转化为内部讲解文档
- `ppt-layout`：将策划方案转化为PPT排版指南
- `exhibition-theme-brainstorm`：展陈主题推导

## 输出示例

生成的展陈大纲通常包含：

- **第一页**：展陈主题（如"无界触达"）、副标题（如"瞬间即永恒 此刻即未来"）、主题解读、核心理念
- **第二页**：核心问题与答案、叙事结构（如"六幕式情感旅程"）、空间规划（6个区域）、情感曲线
- **第三页**：6个区域的详细大纲，每个区域包含2-4个核心模块，每个模块说明展示形式、核心信息、叙事功能

## 注意事项

1. 确保从策划方案中准确提取展陈主题和核心理念
2. 理解叙事逻辑，确保情感曲线合理
3. 空间规划要符合实际空间限制
4. 每个模块的展示形式要明确，便于后续设计
5. 叙事功能要清晰，说明每个模块在整体叙事中的作用
