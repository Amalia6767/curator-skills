# 验证报告模板

输出验证报告时，按以下结构生成：

```markdown
# 内容验证报告

> 验证时间：{日期}
> 待验证文档：`{文件路径}`
> 原始资料：{逐行列出原始资料路径}

## 验证结论

| 指标 | 结果 |
|------|------|
| 事实性陈述总数 | {N} 条 |
| ✅ 有据 | {n1} 条 |
| ⚠️ 推断 | {n2} 条 |
| ❌ 无据（疑似幻觉） | {n3} 条 |
| 🔄 变形（信息偏差） | {n4} 条 |
| 准确率 | {n1/N × 100}% |

## 总体评级

{根据以下规则给出评级}

- 🟢 **可信**：准确率 ≥ 95%，无"无据"项
- 🟡 **基本可信，需复核**：准确率 ≥ 85%，或存在少量"推断""变形"
- 🔴 **存在风险，需大幅修订**：准确率 < 85%，或存在"无据"项

评级：{评级}

## 高置信度问题（两轮验证均发现）

{如无则写"无"}

### 问题 1
- **原文摘录**：{待验证文档中的原文}
- **问题类型**：{数字错误 / 名称错误 / 无中生有 / 含义偏差}
- **原始资料原文**：{对应原始资料内容}
- **说明**：{问题解释}
- **建议修改**：{修正后的表述}

## 待确认问题（仅单轮验证发现）

{如无则写"无"}

### 问题 1
- **原文摘录**：{原文}
- **发现方**：{溯源标注 / 交叉验证}
- **问题描述**：{描述}
- **另一方结论**：{另一轮验证对此的结论}

## 溯源标注明细

| # | 陈述摘要 | 状态 | 出处/备注 |
|---|----------|------|-----------|
| 1 | {摘要} | ✅/⚠️/❌/🔄 | {原始资料中的对应位置或说明} |
| 2 | ... | ... | ... |
```
