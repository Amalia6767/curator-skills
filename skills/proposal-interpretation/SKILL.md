# 策划方案诠释文档生成

## 目标

将策划方案转化为内部讲解文档，让团队内部其他人员（设计师、项目经理、销售等）快速理解：
- **核心理念**：展陈主题、设计哲学
- **核心创意**：关键亮点、差异化优势
- **核心逻辑**：业务理解、叙事结构
- **核心价值**：对甲方的价值、对项目的意义

## 输入

- 策划方案（Markdown格式，通常来自 `proposal-design` skill）
- 可选：展陈主题推导报告

## 输出格式

生成 `策划方案_诠释版.md` 文件，包含：

1. **一句话核心**：用一句话概括整个方案
2. **三大核心**：核心理念、核心创意、核心逻辑
3. **关键亮点**：3-5个最值得强调的创意点
4. **讲解脚本**：适合口头讲解的简洁版本

## 诠释原则

### 1. 简洁有力
- **一句话说清楚**：每个概念用一句话解释
- **避免术语堆砌**：用通俗语言解释专业概念
- **去除冗余**：只保留最核心的信息

### 2. 理念清晰
- **逻辑链条**：理念→创意→实现的完整链条
- **因果关系**：为什么这样设计，解决了什么问题
- **价值主张**：对甲方、对项目、对团队的价值

### 3. 创意明确
- **具体可感知**：创意要能说清楚、能想象
- **差异化突出**：与竞品的区别在哪里
- **可执行性**：创意如何落地

### 4. 避免冗长
- **每段不超过3句话**：保持简洁
- **用表格/图表替代长段落**：视觉化表达
- **提供"一句话版本"和"详细版本"**：按需选择

## 工作流程

### 步骤1：提取核心
1. 提取展陈主题（一句话）
2. 提取核心理念（3-5个关键词）
3. 提取核心创意（3-5个亮点）
4. 提取核心逻辑（业务理解+叙事结构）

### 步骤2：组织诠释
1. **一句话核心**：用一句话概括
2. **三大核心**：理念、创意、逻辑
3. **关键亮点**：3-5个最值得强调的点
4. **讲解脚本**：适合口头讲解的版本

### 步骤3：优化表达
1. 检查是否简洁（每段不超过3句）
2. 检查是否清晰（逻辑链条完整）
3. 检查是否明确（创意可感知）
4. 检查是否冗长（删除冗余）

## 输出结构

```markdown
# [项目名称] 策划方案诠释版

## 一句话核心
[用一句话概括整个方案的核心]

## 三大核心

### 1. 核心理念
[展陈主题 + 设计哲学，一句话解释]

### 2. 核心创意
[3-5个关键创意点，每个一句话]

### 3. 核心逻辑
[业务理解 + 叙事结构，一句话解释]

## 关键亮点（3-5个）

### 亮点1：[名称]
- **一句话**：[一句话解释]
- **为什么重要**：[一句话说明价值]
- **如何实现**：[一句话说明落地]

### 亮点2：[名称]
...

## 讲解脚本（5分钟版本）

### 开场（30秒）
[一句话核心]

### 核心理念（1分钟）
[展陈主题 + 设计哲学]

### 核心创意（2分钟）
[3-5个关键创意点]

### 核心逻辑（1分钟）
[业务理解 + 叙事结构]

### 收尾（30秒）
[价值主张]
```

## 示例

### 输入（策划方案片段）
```markdown
## 核心策划理念：无界触达

> **展陈主题**：无界触达
> **副标题**：瞬间即永恒 此刻即未来
```

### 输出（诠释版）
```markdown
## 一句话核心
用AI重新定义金融服务的时空边界，让金融服务触达每一个人。

## 三大核心

### 1. 核心理念
**无界触达**：AI重构成本，消除边界，让金融服务触达每一个人。
**瞬间即永恒 此刻即未来**：每一次"即刻"的响应，汇聚成永恒的价值。

### 2. 核心创意
- 数十米数据星河：每一次闪烁代表"此刻"的交易
- 一键切换：A/B/C风格秒切换，无界体验
- 数智足迹：体验延续馆外，持续触达
- 千人千面：四类访客，精准适配

### 3. 核心逻辑
AI重构成本→消除边界→让金融服务触达每一个人
```

## 注意事项

1. **避免术语堆砌**：用通俗语言，避免"DNA五力"、"Transformer理念"等术语
2. **强调价值**：每个创意都要说明"为什么重要"
3. **可执行性**：创意要能说清楚如何落地
4. **适合讲解**：提供口头讲解的脚本版本
