# 策划方案诠释文档生成 Skill

本 skill 用于将策划方案转化为内部讲解文档，让团队内部其他人员（设计师、项目经理、销售等）快速理解核心理念与思路。

## 文件结构

```
proposal-interpretation/
├── SKILL.md          # 核心技能说明与工作流程
└── README.md         # 本文件
```

## 输入

### 必需输入
- **策划方案**（Markdown格式）：通常来自 `proposal-design` skill 的输出
  - 例如：`投标方案_策划篇.md`

### 可选输入
- **展陈主题推导报告**：用于理解主题的推导逻辑

## 输出

生成 `策划方案_诠释版.md` 文件，包含：

1. **一句话核心**：用一句话概括整个方案
2. **三大核心**：核心理念、核心创意、核心逻辑
3. **关键亮点**：3-5个最值得强调的创意点
4. **讲解脚本**：适合口头讲解的简洁版本（5分钟版本）

## 使用场景

- **内部培训**：向新加入的团队成员讲解方案
- **跨部门沟通**：向设计师、项目经理、销售等传达核心理念
- **快速汇报**：向领导或客户快速传达核心思路
- **方案传承**：确保团队对方案的理解一致

## 工作流程

1. **提取核心**：从策划方案中提取主题、理念、创意、逻辑
2. **组织诠释**：按照"一句话核心→三大核心→关键亮点→讲解脚本"的结构组织
3. **优化表达**：确保简洁、清晰、明确、不冗长

## 诠释原则

- **简洁有力**：一句话说清楚，避免术语堆砌
- **理念清晰**：逻辑链条完整，因果关系明确
- **创意明确**：具体可感知，差异化突出
- **避免冗长**：每段不超过3句，用表格/图表替代长段落

## 与其他 skill 的关系

- `proposal-design`：生成策划方案
- `proposal-interpretation`：将策划方案转化为内部讲解文档
- `ppt-layout`：将策划方案转化为PPT排版指南
- 三者配合使用，完成从方案到讲解到设计的完整流程
